# AGENTS.md

## Must-follow constraints

- **Prisma Client output** is `generated/prisma` (gitignored). Import from `../../generated/prisma/client` (or relative equivalent), not `@prisma/client`.
- `PrismaService` uses the **pg adapter** (`@prisma/adapter-pg`) — do not replace with the default Prisma connection. See `src/prisma/prisma.service.ts`.
- The FastAPI ingest service URL env var is **`INGEST_ENPOINT`** (typo is canonical). Do not rename.
- Stripe webhook endpoint (`POST /api/subscription/webhook`) requires **raw body** — `bodyParser.raw()` is registered for this path in `main.ts` before `bodyParser.json()`. Do not reorder.
- All API routes are under the global prefix `api` (set in `main.ts`).
- WebSocket gateway runs on **port 3002**, namespace `events`.
- Auth guards: use `AccessTokenGuard` for regular protected routes, `JwtGuard` (from `google.guard.ts`) for subscription routes, stack `AccessTokenGuard + AdminGuard` for admin routes. Both `AccessTokenGuard` and `JwtGuard` extend `AuthGuard('jwt')` but are used in different modules — follow existing pattern per module.
- Email templates have `_tr` suffixed variants for Turkish. Both `en` and `tr` templates must be updated together.

## Validation before finishing

```bash
npx tsc --noEmit            # tsconfig has strictNullChecks: false, noImplicitAny: false
npm run build               # nest build — compiles to dist/
npm run lint                # eslint — errors must be zero
npx prisma generate         # regenerates generated/prisma after schema changes
```

## Repo-specific conventions

- **Module structure**: each feature has `{name}.module.ts`, `{name}.controller.ts`, `{name}.service.ts`, optional `dto/` folder. New modules must be imported in `app.module.ts`.
- **User ID extraction** in controllers: `req.user?.sub || req.user?.id` — JWT payload uses `sub` for user ID.
- `IUser` interface (`src/util/interfaces.ts`): `{ sub, email, iat, exp, teamId, role }`.
- Soft deletes: models use `isDeleted: Boolean` + `deletedAt: DateTime?`. Filter by `isDeleted: false` in queries.
- DB triggers in `prisma/triggers.sql` (`pg_notify` for `Storage` and `Content` updates) must be applied manually — Prisma migrations do not manage them.
- `nest-cli.json` copies `.hbs` and `.html` assets — add new template extensions to `compilerOptions.assets` if needed.
- MinIO file path: `{userId}/{botId}/{fileType}/{hashedFileName}{ext}`.
- Admin settings (token limits, pricing) are stored in `SystemSettings` table, not env vars.

## Important locations

- `src/prisma/prisma.service.ts` — PrismaClient with pg adapter (all services inject this)
- `src/authentication/utils/` — all auth guards and passport strategies
- `src/templates/` — email HTML templates (en + tr variants)
- `prisma/triggers.sql` — PostgreSQL triggers for real-time notifications (must be applied manually)
- `.env.stripe.example` — only env example file that exists; no general `.env.example`
- `generated/prisma/` — gitignored, auto-generated by `prisma generate`

## Change safety rules

- Schema changes require: edit `prisma/schema.prisma` → `npx prisma migrate dev` → `npx prisma generate`. Missing generate causes import errors from `generated/prisma`.
- Do not change the `bodyParser` middleware order in `main.ts` — Stripe webhook signature verification depends on raw body being parsed before JSON.
- `fastapi_ingestiontask` model in Prisma maps to a table owned by the FastAPI service. Do not add migrations that alter this table.
- FastAPI integration uses `HttpService` (Axios). Payloads use `bot_cuid` / `customer_cuid` (snake_case), not camelCase.

## Known gotchas

- `INGEST_ENPOINT` typo is used across `content.service.ts`, `file.service.ts`, `bot.service.ts` — changing it requires updating all env files and deployments.
- `JwtGuard` is exported from `google.guard.ts` (not a dedicated file) — don't create a separate `jwt.guard.ts`.
- `RolesGuard` verifies JWT manually with `process.env.JWT_SECRET` instead of using `ConfigService` — maintain this pattern for consistency.
- `src/lang/` translation dictionaries are empty stubs. Email localization is done via separate `_tr.html` template files, not through the `Language.translate()` method.
- `schema.prisma.back` in project root is a backup file — ignore it.
- TypeScript strict mode is off (`strictNullChecks: false`, `noImplicitAny: false`) — the codebase uses `any` extensively.
- When building Docker images on Apple Silicon (arm64) for AWS/Kubernetes amd64 nodes, build with `--platform linux/amd64` to avoid `no match for platform in manifest` / `ImagePullBackOff`.
